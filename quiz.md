1) What does API stand for?
Application Programming Interface

2) How would you describe an API in your own words?
A tool that allows your code to talk with (use the goodness from) someone else's code (Web APIs, 3rd-party package, etc)

3) Can you think of an example of an API you've used?
* BoredAPI - https://www.boredapi.com/ (getting data from a server)
* Local Storage (localStorage)

4) What is a client?
Any device that connects to the internet to get data from somewhere (makes a "request")
Laptop, phone, tablet, any "smart" device (watch, doorbell, robot vacuum, even a server)

5) What is a Server?
Basically just a computer.
Accepts requests from clients asking for something, then responds to the client with that thing (eg, an HTML page, an image or file, or just plain data)

6) What are some examples of "clients" you've used today?
Laptop
Smart Phone
Robot Vacuum
smart plugs

7) How would you explain what a "server" is to a 5 year-old
A computer that send my own computer things when I ask it to.

8) In what way do clients and servers interact with each other?
Client sends a request to a server, and the server sends back a response.

9) What is a Request?
When a device asks for a "resource" (data, image, HTML page, authentication token, etc)

10) What are 3 things your computer (client) might request from a server?
* JSON array of suggested videos
* Video stream
* HTML page

11) what is the main job of a server?
receive a request from a client and return a response
